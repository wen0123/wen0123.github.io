文件整理的目的，始终是快速找到自己所需的文件

曾经有人向我发出了这样的疑问——
既然通过搜索就可以快速找到文件，那为什么还要整理文件？
我当时给他打了一个比方：
你能找到家里每一件物品，但是你会把食材放在卧室里吗？
桌面毕竟是桌面，不可能把所有文件都放在桌面上管理，总要用到硬盘中的文件。文件存放的时间越长，对文件位置的记忆出现偏差的概率就越高，寻找这些文件所消耗的时间就会越长。
而每一次在硬盘中 反复寻找文件 的过程，都在影响工作效率。
通过整理文件，可以有效地简化步骤、减少时间，哪怕一时想不起文件名，也不会被「放在厨房的食材」困扰。

主要内容介绍

在第二章节的「管理常用文件」中，我将依次介绍以下的内容——
  • 找出你最常用的文件：在日常生活中，最困扰你的往往不是那些乱成团的历史文件夹，而是那些历史文件夹中存放的 常用文件。每一次要用到这些文件，都需要进入 n 层目录去找。如果碰到了这种情况，试试把这些常用的文件都找出来吧。
  • 利用好 Windows 的快速访问：Windows 的 快速访问 是一个非常实用的功能，任何文件夹都可以放进 快速访问 。增删简便，打开迅速。无论是每天都要使用的目录，还是临时存放的文件夹都可以固定在这里。再也不用担心工作被中途打断了。
  • 利用好 Windows 的库，建立工作流：如果快速访问不能满足需求；如果对工作流程很明确；如果工作的系统性很强——那就开启 Windows 的 库 功能吧！ 库 是文件夹的集合，当工作所需的文件关联程度高、文件类型杂时，利用库整理出一套适合自己的工作流，真正意义上提升工作效率。
  • 将软件放在开始屏幕：除了管理常用的文件，常用的软件亦不容忽视。前一章中的「Dock 栏，给软件找一个归宿」，将一些软件固定在了底部的菜单栏。可是电脑中有那么多软件，Windows 又没有 macOS 那么方便的 Launchpad，应该如何管理这些软件呢？其实 Windows 10 的 开始菜单 ，就提供了管理软件的便利。

（一）找出你最常用的文件

通过阅读这一章节的内容，可以找到你经常使用的文件，以便更好地使用它们。
适用人群：轻量级办公用户，反复用到某一文件的用户。

在整理文件之前……

先来看一个使用场景——
Lily 想打开一张财务报表，先打开了文件资源管理器，进入 D 盘，依次打开 公司财务 → 5月财务情况文件夹，再找到 5月财务报表.xls 。双击打开，开始编辑文件。
编辑到一半，Lily 想起同事 Lee 托她发一份名单给他。于是重新打开 D 盘，进入工作文件 → 人员调动情况 文件夹，再打开 5.16 最新部门人员清单.xls ，再拖到 QQ 上发送给 Lee。
处理完财务报表之后，在另存为的界面进入 D 盘，依次打开 数个文件夹……
这其实就是大部分人的生活。我们经常需要不断地在茫茫多的文件中找到特定的文件，在文件资源管理器中跳转、寻找——更何况这还是整理的比较好的情况。更多没有文件整理习惯的人，硬盘的根目录下可能堆积了大量文件夹，每一次寻找文件都显得异常痛苦。
有时候这些常用文件并不需要时时刻刻都极度严格地归类到文件夹中。就拿书桌举例，最常用的文件总是放在桌上、抽屉里，没有谁会把经常用的笔、鼠标、数据线这些东西每次用完都放进书架、放进箱子，这样找起来实在太麻烦了。

书桌
在上一章中，已经整理完了桌面。如果要给常用文件找一个地方，那就——
找一个抽屉，把常用的文件放起来。

为什么要先管理常用文件？

整理文件仓库并不是一件简单的事情，注定需要花费时间和精力。历史文件越多，这项工作就越烦杂。而且短时间内难以看到成效，很容易半途而废。
如果单纯地强调文件的 整理 而不注重 管理。则很容易脱离工作环境，出现上面这种反复跳转、反复寻找的情况。即使花了很多精力整理完文件，也很难对工作效率有所提升。
所以我将「管理常用文件」放在了「管理文件仓库」之前。毕竟 整理抽屉 ，要比整理书架、整理箱子要方便得多，见效快得多。

找出最常用的文件并发送到桌面

无论是生活还是工作，总会有一部分文件经常使用到。这个 经常 可能是每次工作都需要用到，也可能是近期经常用到的文件。
它们可能是各种软件、自己最近在阅读的 PDF 书籍、正在撰写的文稿、一些工作上用到的素材、名单。
有时候可能连自己都没有意识到，这个文件居然用了那么多次。那么，从现在开始就可以留意一下，你经常用到的是哪些文件。
除了自己平时注意，还可以利用 Windows 提供的 最近使用的文件 功能。它的的筛选标准是「在近期 打开 过的非可执行文件」。也就是说打开软件、复制等操作不会被记录，很适合找出 常用文件 。

快速访问的界面
最后将它们的快捷方式发送到桌面，即可在短期内快速访问，不必无数次的打开目录了。
具体操作
  1. 在此电脑 → 快速访问 → 最近使用的文件 中查看你近期访问过的一些文件和文件夹。
  2. 如果你在这个列表中发现了不适合展示出来的内容，可以在左上角的文件 →更改文件夹和搜索选项 → 常规 → 隐私 中选择是否显示常用文件和常用文件夹，也可以清除这里的历史记录。
    文件夹选项的界面
  3. 在这些文件中挑选一些经常使用的文件，鼠标右键 → 发送到 → 桌面快捷方式。就可以在桌面上看到这个文件的快捷方式了。
    快速访问界面
Tip：如何在 Windows 7 中开启「最近使用的文件」
具体操作：右键任务栏开始图标 → 属性 → 开始菜单 → 勾选「存储并显示最近……打开的程序、项目」 → 自定义 → 自定义开始菜单 → 勾选最下方的「最近使用的项目」 → 确定。
就可以在开始菜单中找到「最近使用的项目」了。

为什么使用快捷方式而非复制

我发现有不少人会将常用的文件随手拖到（复制到）桌面上，这种行为其实是不可取的。
在复制文件之后，每当对源文件或者对桌面上的文件进行修改，就会导致同一个文件的产生新、老两个不同的版本。更让人头疼的是，这两个文件存放的位置不同。如果该文件需要打印或发送，一旦错发了较旧的文档，还会引发更多的麻烦。
而 快捷方式 本质上是一个路径，指向了文件本身。如果把电脑比作一本书，快捷方式就是目录上的 第 25 页 ，我们实际上打开、查看的，都是第 25 页的文章内容。
将快捷方式放在桌面上，就保证了我们打开、查看、修改的永远是 同一份文件 ，不用考虑「这份文件是什么时候修改的」。当工作结束，桌面上的快捷方式可以随时删除，原稿则一直存放在硬盘中。

快捷方式与复制的对比

如何分辨快捷方式

请认准这个红色框内的小箭头，带有这个小箭头的文件都是快捷方式。

快捷方式
有些美化软件为了美观会去掉这个小箭头。可以 右键点击文件 查看 属性，文件类型会显示为 快捷方式 (.lnk)。

灵活运用快捷方式

值得注意的是，在复制文件和发送文件的时候，不能直接把这些文件发送给对方，否则对方是无法打开的。你需要 右键点击快捷方式 → 打开文件所在的位置 ，然后把它发送给其他人。
打个比方，想把书中某一章的内容分享给别人，只把目录撕下来显然不行，一定要把第 99 页的内容撕下来。
由于快捷方式不适合分享，所以小文件、正在处理的文件可以直接放在桌面上，经常用的 LOGO 图标。不要采用快捷方式的形式。
无需分享 的大文件作为快捷方式。比如刚刚下载的「某电影.mp4」，又比如最近正在拍摄的一个巨大的「视频项目」文件夹，都可以发送到桌面上，便于快速地访问。
快捷方式这种方式适合轻量级办公的用户，少量常用文件完全可以放在桌面上自由调用。
不过，当常用的文件数量很多的时候，这个方法就不太合适了，好不容易整理好的桌面，怎么能让快捷方式重新占领呢？
下一节就介绍更通用的「快速访问」功能。

适用人群：经常访问特定文件夹的用户，不愿意在桌面放太多图标的用户。

如果你一时无法下定决心完成复杂的整理工作，更方便快捷的「快速访问」或许能帮到你。

如何使用「快速访问」

在 Windows 10 中，对文件资源管理器的一个重大改进就是「快速访问」。打开文件资源管理器，左侧导航栏的最上方，就是快速访问的位置。
「快速访问」分为两个部分，常用文件夹、最近使用的文件。从名字上就可以看出，它的作用就是方便我们快速访问近期的文件。
「快速访问」并不是一个文件夹，更像是本地文件版的收藏夹。在使用时有两种方式。

快速访问

方式一：自动记录常用文件、文件夹

「快速访问」可以自动记录常用的文件和文件夹。上一章就提到过，它的的筛选标准是「在近期 打开 过的非可执行文件」。只要通过文件资源管理器打开的文件，都会被记录在这里。
这个功能非常适合文职工作者，比如在完成这份文稿之后，我还要对文章内容进行补充和修改。我无需做任何动作，就能在「快速访问」中打开这篇文稿。
不过文件夹的自动记录就没有那么清晰，触发自动收藏的条件不明确。有时候打开一个文件，文件夹就会被记录下来。有时候无论是反复进入同一个文件夹，还是打开文件夹中的所有文件，都不会被记录。在文件夹方面建议手动添加。

方式二：手动添加文件夹

「快速访问」中支持手动添加文件夹，不支持添加单个文件。
具体操作 ：
右键任意文件夹 → 固定到快速访问。也可以直接把文件夹拖进「快速访问」。

固定到快速访问
值得注意的是，「快速访问」不仅支持本地文件夹，还支持 OneDrive、WebDAV 等服务器上的文件夹。

快速访问与快捷方式，哪个更适合我

快速访问 类似于浏览器中的收藏夹，与单一的 快捷方式 有所不同。快速访问用于有组织、明确的文件分类。
快速访问位置显眼，它将常用文件夹集结在一起，适合工作文件丰富、近期文件集中的用户使用。便于在整个文件库中进行快速访问；可以自动记录最近的文件夹和文件，可以访问支持 WebDAV 和 FTP 的远程服务器。
快捷方式 是零散、自由的。可以根据自己的喜好随意放置，用完即删，不必考虑后果。
可以将一个文件的快捷方式发送到多个文件夹中，如果电脑 C 盘空间紧张，靠快捷方式可以大大缓解桌面大文件造成的压力；可以在路径后添加后缀，可定制不同的功能，比如在路径最后加上 -win 就能强制窗口化。但是缺点也很明显，它记录的是绝对路径，只适合在本地电脑上使用，且容易被误发给别人。
结论如下——
当文件数量不多、需要完成一些轻量工作时，多用快捷方式。
当需要集中力量完成一个项目时，将项目文件夹固定在快速访问中。便于有组织的访问文件。

快捷方式和快速访问

如何保证自己的隐私

自动记录虽方便，但是也会涉及到隐私文件的泄露。很多人并不喜欢把最近使用过的文件展示在「快速访问」中，在第一时间就关掉了自动记录功能，甚至修改注册表将其彻底关闭。但实际上我们又确实需要这样一个功能。
那么有什么办法既能使用「快速访问」又能确保自己的隐私呢？
  1. 可以在设置打开文件资源管理器时，默认显示界面为「此电脑」。这样只有当需要时才会看到近期文件。
  2. 在结束工作时，收起「最近使用的文件」的标签。
  3. 使用第三方的文件资源管理器的收藏夹功能，比如 XYplorer、Total Commander 等优质资源管理器。在后面的章节中也会进行介绍。

修改默认位置
如果觉得这些方法都不适合自己，Windows 中还提供了更严谨、更专业的「库」来管理自己的常用文件。

（三）利用好 Windows 的「库」

通过阅读这一章节的内容，你可以学到 Windows 自带的「库」的使用技巧。可以对电脑中不同位置的文件夹进行统一归纳，形成一套文件的使用流程。
适用人群：适合身负一些重要项目，需要一套完整工作流程的用户。

利用「库」收纳同类型的文件

前面两节的做法适用于「文件较少的轻量级用户」，当我们处于生产环境下，文件会变得相当复杂，短时间内会用到大量杂乱的文件，包括资料、素材、附件……个人文件和工作文件之间也会不可避免的混杂起来。
把它们作为快捷方式发送到桌面不合适，我们经常会使用、查看、转发这些文件，谁也不想把快捷方式发送给别人。而且它们通常不是同一类文件，直接放在同一个文件夹中又显得太杂乱。
所以当你有大量常用的文件，不建议将它们聚集在到一起，因为这会导致文件在积累到一定数量之后放弃整理，使文件层级更加混乱。
那么究竟应该怎么办呢？
归纳到「库」中。
尽管 Windows 中没有提供 Mac OS 下的标签功能，但是 Windows 提供了另一个思路的整理方法，不需要单独标记 每一个 文件，而是将 同一类 文件归纳到一起。
接下来就介绍一个更高效快捷的整理归纳的方法——这就是 Windows 中提供的「库」功能。

第一步：开启「库」功能

在 Windows 10 中，微软在新增「快速访问」的同时，也取消了「库」的默认显示。这是微软态度的转变。快速访问门槛低，逻辑容易理解，适合大众用户使用。而「库」更符合深度用户的需求，需要的用户自然会去开启这项功能。
「库」在 Windows 10 中默认是关闭的，你可以在上文中提到的 文件夹选项 中，勾选 显示库。

显示库
开启「库」之后，你可以在文件资源管理器的左侧边栏中看到新出现的「库」。

第二步：了解「库」与默认文件夹的区别

「库」和「此电脑」中的默认文件位置很容易混淆。不仅默认名称一样，不细看图标也很难看出区别。那么两者的区别是什么呢？

库与默认文件夹的区别
  • 「此电脑」中的「视频」、「图片」、「音乐」，都是存在于「C:\Users\用户名\」目录下的普通文件夹，和其他的文件夹没有本质区别。同时由于很多软件会将其作为默认的保存位置，并且占用的是 C 盘空间，为了防止混乱，也不建议把自己的文件放在这里。
  • 「库」更像是一个「文件夹的集合」，你可以将同类型的文件夹归类在一起，并且在同一个界面中访问所有文件夹中的文件（普通文件夹只能一个一个打开查看），这也保证了硬盘中的文件不会混杂。
此处的「库」与 Mac OS 的「标签」有所区别，「库」的操作不影响文件本身，是独立于文件外的一种归类方式。

第三步：新建一个「库」

具体操作：
  1. 在库的图标上 右键 → 新建 → 库。
    新建库
  2. 右键新建的库 → 属性 → 添加 → 选择文件夹 → 确定 。就可以将你的多个文件夹添加到这个库中。
    添加库文件
  3. 调整文件夹的顺序。

第四步：利用「库」打造自己的 WorkSpace

新建库的操作没有难度，学会了如何建立新的库，可以试着产生自己的 WorkSpace 了。
一种分类方式是按照工作文件和私人文件分类。新建「工作库」和「生活库」，分别用于放置工作文件和私人文件。

新建一个默认库
库是有顺序的，所以可以按照自己的工作流进行排序。
如果有很多经常要用到的文件，或者是最近在忙一个项目任务。可以创建一个「项目常用库」，并将项目相关的所有文件夹都放在这个库中，并按照项目顺序进行排序。

将「库」与工作流程结合

举个例子，Anika 参与了一个开发项目，负责其中的 UI 和部分代码。
在这个项目中，Anika 有以下这些需求：
  1. 首先需要查看该项目的相关文档资料，需要经常浏览 文档库中的项目文档。
  2. 之后需要收集一些素材和关于 iOS UI 的设计标准，需要 素材库 和 UI 设计标准 。
  3. 然后需要完成代码部分的编写，需要 CodeLibrary 。
  4. 偶尔也会需要在 Windows 下使用 Mac OS 的虚拟机，需要用到 Mac 虚拟机。

常用库
Anika 将以上所有需要用到的文件夹，全部放在一个库中，以便这段时间方便快捷的使用这些文件。
这样做的一个优势，在于硬盘中的文件依然是你之前的文件，使用「库」不需要对你电脑中的文件做任何调整。在查看这些文件的时候，再也不需要深入到每一个文件夹，打开自己需要的文件了。

库之间移动
「库」的另一个优势在于，它不仅可以按照你自己的顺序对文件夹进行排列，还可以在文件夹之间快速进行拖放操作。相比在多个窗口之间进行拖动，这样操作在效率上会有很大的提升。

利用「库」完成下载内容的收纳

除了更高效率的工作流程，再来看看每个人都会碰到的问题——下载。
虽然下载这件事在网速大幅提升的今天已经不是什么难事，不过你能快速找到你之前下载的文件吗？
比如迅雷会在剩余空间最大的磁盘中自动创建一个 迅雷下载 文件夹，QQ 旋风会在 D 盘默认创建一个 QQDownload，各种浏览器可能会自动创建一个 搜狗高速下载、360Download，一些手机管理软件会创建各自的 XXDownload 目录。哦对了，还有系统默认的下载位置：C:\Users\用户名\Downloads。
面对如此多的目录，除非你第一次下载完文件就打开它，那么第二次想找这个文件的时候很可能已经忘了它在哪里了。
这个时候我们就可以利用「库」来实现下载内容的收纳。
具体操作：
  1. 按之前的新建方法，新建下载库。
  2. 如果你比较懒，可以直接隐藏电脑中所有的与 Download 相关的文件夹。当然最好的方法是将每个软件的默认下载路径修改到统一的「下载」目录中。
  3. 将电脑中所有的与 Download 相关的文件夹，全部添加到新建的「下载库」中。
  4. 在这个库中，右键 → 分组依据 → 更多 → 找到 访问日期 并勾选 → 确定后重新右键 → 选择按照访问日期分组。
完成这几个简单的步骤。你就会发现电脑中所有的下载内容都按照时间分组了。从此寻找历史下载文件只需点击一次。

下载库
至于第四步中为什么要按照 访问日期 而不是默认的 修改日期 分组，那是因为有些文件在下载时会保留文件本身的修改日期，图中的 adbdrive for Xiaomi.zip 就是这样，修改日期是 2015 年。按修改日期排列会导致新下载的文件出现在较老的位置，而访问日期基本可以认为是 下载时间，这样排序更合理。
没错，「库」的许多属性，非常适合完成一个项目，通过「库」把同类型文件都收纳在一起，或者把工作流程收纳进库中，可以很大程度上方便我们查找和使用这些文件。
只要好好利用「库」，它一定能为你的工作效率带来突破性的提升。

（四）XYplorer：更优质的资源管理器

通过阅读这一章节的内容，可以学习利用 XYplorer 对文件进行更高效的管理，实现更高级的过滤、筛选、自定义快捷键、脚本处理等操作。

XYplorer 配置

XYplorer 入门

考虑到 XYplorer 功能非常多，本文无法做到面面俱到。我挑选了一些日常用得到的操作，供各位快速上手这款软件，可以在短期内提升效率。

多种快捷浏览功能

浏览是文件资源管理器最重要的功能之一，如何提升浏览效率自然是需要首先考虑的。那么 XYplorer 又在哪些地方改进了浏览体验呢？
多标签页 + 双面板 F9 ：按住 Shift 并 单击 文件夹，可以打开一个新的标签页。 按住 Alt 并 单击 文件夹，可以打开一个新面板。尽管多标签页和双面板都不是什么新鲜的概念。但是 XYplorer 将两者结合了起来，让浏览文件和移动文件更为高效。
浮动浏览 F11：这个功能类似 Mac OS 中的「空格浏览」功能，不打开文件就可以看到文件内容，按键也可以自定义。（这个功能也可以用一款名为 Seer 的软件单独实现。）
迷你树视图 ：硬盘里有那么多文件，需要用的往往是几个特定的目录。先打开常用目录后开启迷你树视图，左侧的导航栏就会只留下当前的目录，排除了多余项的干扰。
展开浏览 ：当一个目录很深，比如 D:\文档库\付费内容稿件\如何整理我的 Windows 文件\02-管理常用文件\图片\2-4-1-XYplorer配置.png，需要打开 6 层文件夹。但是在展开浏览视图下，可以直接看到某个目录下包括所有子文件夹内的所有文件。

迷你树视图

文件过滤系统

XYplorer 中的文件筛选过滤系统完善，提供了一些常用的筛选器，可以快速筛选出各种类型的文件。结合上文提到的 展开浏览 可以迅速筛选出子文件夹下的所有文件，会自动按照路径层级排序，非常实用。
如果不满足于简单的过滤，XYplorer 还支持用户自行设定的文件类型 (单种或多种拓展名)，可以筛选出特定类型的文件以便批量操作。

优化过的文件操作

移动和复制，是最常用的两个操作。默认的资源管理器一旦出错就会直接停止所有任务，导致部分文件损坏，需要人工检查才能发现问题。
在 XYplorer 中，不仅完整重绘了操作对话框，还提供了更合理的处理方式。比如在移动过程中碰到重复文件，XYplorer 不会像资源管理器一样提供模棱两可的选项，而是提供更直观的显示处理方式 ：覆盖、增量命名、跳过、忽略等。
还支持多达 256 步的撤销操作、更科学的重命名方式、批量命名文件等细节上的优化。

更多文件操作

XYplorer 进阶

在熟悉了 XYplorer 的基本操作之后，还有一些更有意思的操作。

自定义命令

由于在 XYplorer 中完全没有前文的「快速访问」和「库」，我在写这一篇的时候内心还是很忐忑的。直到我发现它提供了自由度更高的「自定义命令」。
在菜单栏的 工具 → 用户命令 中，可以自定义绝大部分需要的操作。比如 打开工具库，可以对其路径进行自定义，按时间、数量等方式建立文件夹，还可以为这个动作自定义快捷键（图中就是自定义的快捷键）。这些复杂的操作需要了解一些简单的语法，在一旁的 技巧 中都会有所提示。

自定义命令

可替代「库」的纸文件夹

如果说自定义命令的门槛较高，那么 纸文件夹 应该属于简单易懂的那种。纸文件夹实际上是把文件目录存储在一个 txt 文件中，在需要的时候打开它。实现与「库」类似的效果。只不过纸文件夹是存储在本地文件中。

脚本处理

脚本功能的存在意味着这是一个可编程的文件管理器，可以实现自动处理任务。
开启 目录 功能，在左下角会出现一个小框，脚本可以在这个位置快速打开，可以很方便地处理一些任务。比如「生成目录列表」、「自动转化音视频文件」等。（如果只是需要整理文件，我会在第四章推荐一个名为 Droplt 的文件整理软件，比编写脚本更直观。）

是不是每个人都需要用到第三方文件管理器？

并不是，第三方文件资源管理器意味着更高的学习成本，需要熟悉全新的界面和操作逻辑。在一些很小的细节上有所不同就需要重新适应。比如我刚开始使用时就很不习惯 按住 Ctrl + 鼠标滚轮 这个操作由 改变视图 变成了 缩放字号，最后不得不去配置里重新调整。
虽然系统自带的文件管理器实在称不上是「好用」，但它毕竟是系统自带，兼容性、打开速度毋庸置疑。当你离开装了 XYplorer 的 PC，去使用其他 PC 时候，又回归到了当初的操作逻辑。XYplorer 中的标签没了，搜索、筛选这些功能都变得很原始。生疏的操作简直让别人怀疑你是不是新手。
所以在更换原生文件管理器之前，建议体验一段时间，确认它是不是真的能帮到你再做决定。
